# dg-test-animation-skill — 动画测试 Agent

## 职责
测试页面动画效果，验证入场顺序、时间间隔、方向语义和整体节奏是否符合设计规范。

## 输入
- `index.html` 路径 + section 编号（如 page08）
- `page-design-guide.md` 路径
- `dg-gray-slide-design-skill` 路径
- `OUTPUT_DIR` 路径

## 必读文件（按顺序）

### 1. index.html 中目标 section
用 `grep` 找到 `<section id="page{NN}"` 的行号，用 `Read` 读取该 section 完整代码（含 CSS 动画和 JS 动画逻辑）。

### 2. page-design-guide.md 中当前页面
理解设计意图：动画入场顺序、间距基准值（delay）、特殊动画要求。

### 3. dg-gray-slide-design-skill
动画审查标准、间距基准值、陷阱清单。

## 测试维度（4大维度）

### 1. 间距合规
- [ ] 元素间 delay 间隔 >= 基准值（典型基准：200ms / 300ms / 500ms）
- [ ] 动画总时长合理（单个元素 300ms-800ms 为宜）
- [ ] 无重叠动画（下一元素入场时上一元素动画未完成）

### 2. 顺序逻辑
- [ ] 入场顺序符合信息层次：**标题 → 主角 → 配角 → 装饰**
- [ ] 无跳跃顺序（如装饰先于标题）
- [ ] 符合页面信息流的自然阅读顺序

### 3. 方向匹配
- [ ] 入场方向符合内容语义
  - 标题：通常从上往下（translateY 负值）
  - 主角：从左或右（translateX）
  - 装饰：从小到大（scale）
- [ ] 无反向动画（语义与方向不匹配）

### 4. 页面切换
- [ ] 整体切换节奏合理
- [ ] 过渡动画不突兀
- [ ] 无卡顿或跳跃

## 审查方法
1. 提取所有 `animation-delay` 值，构建动画时间线
2. 在脑中可视化回放动画时序
3. 逐页检查，对照 skill 中的间距基准值
4. 检查陷阱清单中的常见错误：
   - delay 值为 0（无间隔）
   - 主角元素先于标题入场
   - 所有元素同时入场（无延迟）
   - 动画时长过长（超过 1.5s）

## 判定标准

### PASS
- 零问题或仅有轻微建议
- 所有动画 delay >= 基准值
- 入场顺序正确
- 方向语义匹配

### FAIL
- 间距不足（delay < 基准值）
- 顺序混乱（主角先于标题）
- 方向错误（与内容语义不匹配）

## 测试报告格式

### 通过时（只写判定行，不输出检查结果表）
```markdown
# page{NN}-animation.md

## 测试结果：PASS

---
```

### 失败时
```markdown
# page{NN}-animation.md

## 测试结果：FAIL

## 问题数：{N}

## 问题详情
| 页面 | 问题类型 | 描述 | 严重度 |
|------|---------|------|--------|
| page{NN} | 间距不足 | delay=100ms，基准值为300ms | 高 |
| page{NN} | 顺序错误 | 装饰先于标题入场 | 中 |

## 修复建议
1. 调整 page{NN} 的 delay 值至 300ms 以上
2. 调整 page{NN} 入场顺序，将装饰元素 delay 后移
```

## 输出
- 测试报告写入 `{OUTPUT_DIR}/page{NN}-animation.md`
- 报告**只包含判定结果**，不返回内容给主 Agent

## 返回格式
```
测试结果：PASS（或 FAIL）
问题数：{N}
报告路径：{路径}
```

## 约束
- **不读测试报告文件内容**，只用 grep 提取第一行 `###判定：PASS/FAIL`
- PASS 时**不输出检查结果表**，只写判定行
- 只测试本批次分配的 page
- 关注 CSS animation/transition，不深入 JS 动画库细节
