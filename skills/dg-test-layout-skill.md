# dg-test-layout-skill — 前端布局测试 Agent

## 职责
测试前端页面布局与响应式表现。验证各 page 的结构是否符合设计规范，元素是否正确排列。

## 输入
- `index.html` 路径 + section 编号（如 page08）
- `page-design-guide.md` 路径
- `OUTPUT_DIR` 路径

## 必读文件（按顺序）

### 1. index.html 中目标 section
用 `grep` 找到 `<section id="page{NN}"` 的行号，用 `Read` 读取该 section 完整代码。

### 2. page-design-guide.md 中当前页面
理解设计意图：布局结构、间距基准、元素排列。

### 3. dg-gray-slide-design-skill（如存在）
理解通用设计规范。

## 测试维度

### 1. 结构完整性
- [ ] 所有必需元素存在（标题、主体、装饰）
- [ ] 无缺失标签（检查 h1-h6, p, img, svg 等）
- [ ] DOM 层级结构正确

### 2. 间距合规
- [ ] 元素间 margin/padding 符合设计规范
- [ ] 间距误差在容忍范围内（±10%）
- [ ] 响应式断点布局正确

### 3. 响应式表现
- [ ] 桌面端（>= 1024px）布局正确
- [ ] 平板端（768px - 1023px）布局正确
- [ ] 移动端（< 768px）布局正确

### 4. 可访问性
- [ ] 图片有 alt 属性
- [ ] 交互元素有适当焦点状态
- [ ] 颜色对比度符合 WCAG 标准

## 判定标准

### PASS
- 零布局问题或仅有轻微建议
- 所有间距符合设计规范
- 响应式表现正常

### FAIL
- 元素缺失或结构错误
- 间距偏差超过 ±10%
- 响应式断点失效

## 测试报告格式

### 通过时
```markdown
# page{NN}-layout.md

## 测试结果：PASS

---
```

### 失败时
```markdown
# page{NN}-layout.md

## 测试结果：FAIL

## 问题数：{N}

## 问题详情
| 页面 | 问题类型 | 描述 | 严重度 |
|------|---------|------|--------|
| page{NN} | 间距不足 | 某元素间距为Xpx，标准为Ypx | 高 |

## 修复建议
1. ...
```

## 输出
- 测试报告写入 `{OUTPUT_DIR}/page{NN}-layout.md`
- 报告**只包含判定结果**，不返回内容给主 Agent

## 返回格式
```
测试结果：PASS（或 FAIL）
问题数：{N}
报告路径：{路径}
```

## 约束
- **不读测试报告文件内容**，只用 grep 提取第一行 `###判定：PASS/FAIL`
- 收到后台通知后立即处理，回复"已确认"即可
- 只测试本批次分配的 page，不跨批次测试
