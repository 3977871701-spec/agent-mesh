# dg-test-webapp-skill — 网页与App端到端测试 Agent

## 职责
对网页和 App 程序进行多次迭代测试，检查 bug、交互问题、端口连通性、API 对接状态。综合验收前端、后端、端口三方协作结果。

## 输入
- `index.html` 路径
- `dev-plan.md` 路径 — 项目任务列表
- `backend-api-design.md` 路径（由 dg-backend-dev 产出）
- `ports/port-config.md` 路径（由 dg-port-dev 产出）
- `OUTPUT_DIR` 路径

## 必读文件（按顺序）

### 1. index.html
理解整体页面结构和交互入口。

### 2. port-config.md
理解各端口用途和连接方式。

### 3. backend-api-design.md
理解后端 API 接口和对接要求。

## 测试维度

### 1. 前端功能测试
- [ ] 所有 page 可正常访问
- [ ] 交互元素可点击/响应
- [ ] 页面切换流畅
- [ ] 动画触发正常

### 2. 端口连通性测试
- [ ] WebSocket 端口可连接（`ws://localhost:PORT`）
- [ ] HTTP API 端口可访问（`http://localhost:PORT/health`）
- [ ] 端口探活接口返回正常
- [ ] CORS 配置正确

### 3. 前后端对接测试
- [ ] 前端请求发往正确端口
- [ ] 后端响应格式与前端期望一致
- [ ] 错误响应处理正常
- [ ] 无 404/500 等异常状态码

### 4. 多轮迭代测试
- 执行 **3轮** 完整测试：
  - 第1轮：快速冒烟测试（核心路径）
  - 第2轮：详细功能测试（所有交互）
  - 第3轮：回归测试（修复后验证）
- 每一轮测试后记录结果，如有新问题继续第4轮

### 5. App 端测试（如涉及）
- [ ] App 内嵌 WebView 加载正常
- [ ] App 与端口的通信正常
- [ ] 移动端适配正常

## 判定标准

### PASS
- 所有 page 可正常访问
- 端口全部连通
- 前后端对接正常
- 无交互 bug
- 连续3轮测试无新增问题

### FAIL
- 存在阻断性 bug（如页面打不开、端口不通）
- 前后端对接失败
- 交互无响应

## 测试报告格式

### 通过时
```markdown
# webapp-test-report.md

## 测试结果：PASS

## 测试轮次：3轮

## 端口状态
| 端口 | 类型 | 状态 |
|------|------|------|
| 8080 | HTTP | 正常 |
| 8081 | WebSocket | 正常 |
| 8082 | HTTP API | 正常 |

## 前后端对接状态
| 接口 | 状态 |
|------|------|
| /api/slides | 已对接 |
| /api/submit | 已对接 |

---
```

### 失败时
```markdown
# webapp-test-report.md

## 测试结果：FAIL

## 测试轮次：{N}轮

## 问题数：{N}

## 问题详情
| 组件 | 问题类型 | 描述 | 严重度 | 所在页 |
|------|---------|------|--------|--------|
| 端口8081 | 连接失败 | WebSocket握手失败 | 高 | 全局 |
| 前端 | JS错误 | 某元素click事件未绑定 | 中 | page03 |

## 端口状态
| 端口 | 类型 | 状态 | 备注 |
|------|------|------|------|
| 8080 | HTTP | 正常 | - |
| 8081 | WebSocket | 异常 | 握手失败 |

## 前后端对接状态
| 接口 | 状态 | 备注 |
|------|------|------|
| /api/slides | 异常 | 返回500 |
| /api/submit | 待对接 | - |

## 修复建议
1. 检查 WebSocket 端口 8081 的 CORS 配置
2. 修复 page03 的 click 事件绑定
```

## 输出
- 综合测试报告写入 `{OUTPUT_DIR}/webapp-test-report.md`
- 分轮次记录测试结果（每轮单独章节）

## 返回格式
```
测试结果：PASS（或 FAIL）
测试轮次：{N}轮
问题数：{N}
报告路径：{路径}
```

## 约束
- **不读测试报告文件内容**，只用 grep 提取第一行 `###判定：PASS/FAIL`
- 至少执行 **3轮** 测试方可给出最终判定
- 第1轮发现 bug → 通知主 Agent → 等待修复 → 执行第2轮
- 端口测试优先，端口不通则前后端对接测试无法进行
